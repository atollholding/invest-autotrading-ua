# ПрАТ "АВТОТРЕЙДІНГ ІНВЕСТ"

Astro + React islands + Sveltia CMS landing site for `invest.autotrading.ua`.

## Commands

```sh
npm install
npm run dev
npm run build
npm run check
```

If local npm has a `before` cutoff in `~/.npmrc`, install can be run without changing global config:

```sh
npm install --userconfig=/private/tmp/npmrc-empty-autotrading
```

## Content And CMS

- Admin UI: `/admin/`
- CMS config: `public/admin/config.yml`
- Navigation/footer content: `src/content/site/navigation.json`
- Company/contact content: `src/content/site/company.json`
- Page content and SEO: `src/content/pages/*.json`
- New issuer document metadata: `src/content/documents/groups/*.json`
- New issuer uploads: `public/documents/uploads/`
- Generated archive manifest: `src/content/documents/manifest.json`

`src/content/documents/manifest.json` is generated by `scripts/build-manifest.mjs`. Do not edit `sizeBytes` or `checksumSha256` manually; the build script calculates them from files in `public/documents/**`.

CMS and staging deployment notes are documented in `docs/cms-and-deployment-plan.md`.
Document archive rules are documented in `docs/document-archive-model.md`.
Commit conventions are documented in `docs/contributing.md`.

## Deployment

Cloudflare Pages:

- build command: `npm run build`
- output directory: `dist`

The site is static and does not depend on Cloudflare-specific runtime APIs, so it can be moved to Netlify, Vercel, or another static host.

Recommended preview flow:

- keep the old site on `invest.autotrading.ua`;
- publish the new site on Cloudflare Pages;
- attach `invest-new.autotrading.ua` for client review;
- switch DNS for `invest.autotrading.ua` only after approval.
